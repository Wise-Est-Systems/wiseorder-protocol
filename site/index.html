WiseOrder Protocol v0.1.0

WiseOrder governs how cognition becomes consequence — the kernel that decides whether an AI's output is allowed to become an action, and seals every decision. One of three peers under Wise.Est Systems.

Built by Henry Wayne Wise III, founder of Wise.Est Systems · read the founder story →

Replayable Deterministic Bounded Verifiable

Same input → same bytes → same result, proven three independent ways (Python, Rust, Go). Every transition is sealed in a tamper-evident, fail-closed audit chain you can verify yourself — offline, no server, no account. That is the concrete floor.

The depth: every AI-proposed action is classified by the kind of claim it makes, judged by the right standard for that kind, and must clear a separate authorization gate before it takes effect — and a judgment can never be recorded as a proven fact. An AI cannot rubber-stamp its own actions. WiseOrder is not a model and not a network service; it is the rules an implementation MUST satisfy.

This site is informational. The authoritative artifacts are in the repository: the specification, the vector suite, the schemas, the conformance report, and the reference implementation.

Demo

The pipeline runtime executes a fully governed workflow: proposer → review gate → bounded executor, with sealed refusal artifacts at every refusal site.

make pipeline-check         # 8 fixtures, both branches
make pipeline-run-fixture   # one canonical valid run, end-to-end

Eight live fixtures cover both branches:

  • Safe: valid pipeline executes admitted commands; produces replayable execution manifest.
  • Unsafe: reviewer rejection, mutated proposal hash, forbidden command, wrong executor identity. Each produces a sealed refusal artifact; the executor is NOT invoked.

Known limitations

  1. No independent implementation. Both declared implementations are first-party.
  2. No external audit. Both implementations report audit_status: NOT_AUDITED.
  3. No production deployment, no operational scars, no adversarial load.
  4. Cross-machine replay verified only on macOS. Linux and second-machine slots are open.
  5. OS isolation is macOS-only in v0.1.0.
  6. Signing is out-of-band. v0.1.0 does not specify a signature mechanism.
  7. Roughly 30% of the spec corpus is backed by executing implementation; the remaining 70% is canon written ahead of code.
  8. Vector suite is breadth-oriented (33 vectors), not exhaustive.

This list is published deliberately. A reviewer who restates one of these items has not found a new finding.

What this is not

  • Not a model. The protocol does not contain a neural network.
  • Not a network service. The reference runtime is single-host with no outbound HTTP.
  • Not a moral system. WiseOrder governs the form a claim must take to be admissible; substantive correctness lives elsewhere.
  • Not a claim of AGI, consciousness, autonomy, or universal trust. Every Non-Goals section in every document says so explicitly.
