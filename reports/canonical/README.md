# Canonical Evidence Index

This directory and its sibling files together define what evidence under `reports/` is **canonical** (load-bearing, must remain stable byte-for-byte under v0.1.0) versus **archived** (historical per-run output, kept for audit but not load-bearing).

## What is canonical

Canonical evidence backs at least one frozen claim in `SPEC_LOCK_v0.1.md`. Modifying it without a protocol version bump is a v0.1.0 freeze violation.

## What is archived

`reports/archive/` contains historical per-run outputs (timestamped JSON+MD pairs). These were generated by past invocations of the pipeline / proposer / review-gate / OS-isolation / resource-limit runtimes. They are kept for audit but are not load-bearing under v0.1.0.

A fresh `make ci` and `make pipeline-check` write new per-run files into the live runtime directories (`reports/pipeline_runtime/`, etc.). Periodically those can be moved into `reports/archive/<runtime>/` to keep the live directories small. **Archiving never deletes.**

## How to verify canonical evidence is intact

```bash
make conformance              # regenerates reports/conformance-{report.json,summary.txt}
python3 -c "import json; print(json.load(open('reports/conformance-report.json'))['vectors_suite_sha256'])"
# expect: sha256:6168d2075931baa98bba763f9cd537142554aae7b26f19e3e93a60de2ee5bb0f

make interop                  # regenerates interop reports
make canonicalization-check   # verifies corpus_sha256
make demo                     # runs all checks + verifies all three frozen fingerprints
```

If any of the three frozen fingerprints diverges, the canonical evidence has been modified incorrectly — investigate before continuing.

## Rule

> Files in `reports/archive/` are historical evidence. They MAY be deleted by an explicit retention policy in a future work order; they MUST NOT be modified. Files in the runtime canonical paths and at the top of `reports/` MUST NOT be deleted or modified except via a fresh run of the canonical commands above.
